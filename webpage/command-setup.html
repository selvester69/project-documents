Create Custom macOS Commands

An interactive guide to turning any shell script into a powerful, global command you can run from anywhere.

1

Create Your Script

A shell script is a text file containing a series of commands. This example creates a powerful `jenkins` command to manage a Jenkins server running in Docker. The first line, `#!/bin/bash`, is called a "shebang" and tells your Mac to use the Bash shell to run the script. 

#!/bin/bash

# Container details
CONTAINER_NAME="jenkins-server"
IMAGE_NAME="jenkins/jenkins:lts-jdk17"

# Function to check container status
check_status() {
  if [ "$(docker ps -q -f name=$CONTAINER_NAME)" ]; then
    echo "Jenkins container is running."
    return 0
  elif [ "$(docker ps -aq -f name=$CONTAINER_NAME)" ]; then
    echo "Jenkins container exists but is stopped."
    return 1
  else
    echo "Jenkins container does not exist."
    return 2
  fi
}

# Main logic
case "$1" in
  start)
    check_status
    status=$?
    if [ $status -eq 0 ]; then
      echo "Jenkins container is already running."
    elif [ $status -eq 1 ]; then
      echo "Starting existing Jenkins container..."
      docker start $CONTAINER_NAME
    else
      echo "Creating and starting new Jenkins container..."
      docker run \
        -d \
        -p 9090:8080 \
        -p 50000:50000 \
        -v ~/jenkins_data:/var/jenkins_home \
        --name $CONTAINER_NAME \
        $IMAGE_NAME
    fi
    echo "Jenkins is running at http://localhost:9090"
    ;;

  stop)
    if [ "$(docker ps -q -f name=$CONTAINER_NAME)" ]; then
      echo "Stopping Jenkins container..."
      docker stop $CONTAINER_NAME
      echo "Jenkins container stopped."
    else
      echo "Jenkins container is not running."
    fi
    ;;

  status)
    check_status
    ;;

  *)
    echo "Usage: $0 {start|stop|status}"
    exit 1
    ;;
esac
2

Make the Script Executable

For security, new files aren't runnable by default. You need to grant "execute" permissions. The `chmod +x` command modifies the file's permissions, telling the system it's safe to run as a program. 

chmod +x jenkins
3

Add it to Your PATH

To run your script from any directory just by typing its name, move it to a location in your system's `PATH`. The `PATH` is a list of directories your shell searches for commands. `/usr/local/bin` is the standard place for user-installed executables.

4

Run Your New Command

That's it! Your new `jenkins` command is ready. Open a **new** terminal tab or window to ensure it's loaded, and try it out. You can now manage your Jenkins container from anywhere.

// Start Jenkins

jenkins start

// Stop Jenkins

jenkins stop

// Check Status

jenkins status
